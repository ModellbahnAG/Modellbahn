\hypertarget{index_Einleitung}{}\doxysection{Einleitung}\label{index_Einleitung}
Platform\+IO ist kein eigenstädniger Editor, sondern ein Plugin, welches für verschiedene Editoren verfügbar ist. Wir nutzen Platform\+IO in Atom. Das ist aber für die Bedienung von {\ttfamily Platform\+IO} nicht wichtig.\hypertarget{platformio_newproject}{}\doxysection{Ein neues Projekt erstellen}\label{platformio_newproject}
Anders als bei der Arduino IDE können wir nicht einfach Platform\+IO öffnen und direkt drauf los programmieren, sondern müssen zuerst ein neues Projekt erstellen. ~\newline
Dafür müssen wir im {\ttfamily  Home Tab } von Platform\+IO {\ttfamily \char`\"{}\+New Project\char`\"{}} wählen und das neue Projekt konfigurieren. Geöffnet wird der {\ttfamily Home Tab} mit dem Knopf  
\begin{DoxyInlineImage}
\includegraphics[width=5%]{Home.png}
\end{DoxyInlineImage}


Jetzt muss das neue Projekt konfiguriert werden. Als Name sollte ein aussagekräftiger Name gewählt werden, damit das Pfojekt auch später wieder gefunden wird. Als Board wird der Typ des Arduinos angegeben, in den meisten Fällen ist das der {\ttfamily Arduino Uno}. Dies lässt sich später noch in der \mbox{\hyperlink{platformio_platformioini}{Konfiguration mit platformio.\+ini}} {\ttfamily platformio.\+ini} Datei anpassen. Als Framework MUSS Arduino ausgewählt sein. Um Github nutzen zu können, muss das Projekt im Ordner {\ttfamily Modellbahn} oder in einem seiner Unterordner gespeichert sein. ~\newline
Sobald das Projekt erstellt wurde, kann in der sich im {\ttfamily src} Ordner befindenden {\ttfamily main.\+cpp} Datei das Programm geschrieben werden.

\hypertarget{platformio_libraries}{}\doxysection{Bibliotheken / Libraries hinzufügen}\label{platformio_libraries}
Die verwendeten Libraries müssen in der \mbox{\hyperlink{platformio_platformioini}{Konfiguration mit platformio.\+ini}} platformio.\+ini Datei eingetragen werden, damit sie verwendet werden können. Glücklicherweise müssen wir das aber nicht selber tun, sondern können dafür die {\ttfamily Libraries} Funktion von Platform\+IO nutzen. Diese verbirgt sich im {\ttfamily Home} Tab hinter dem Knopf  
\begin{DoxyInlineImage}
\includegraphics[width=5%]{Libraries.png}
\end{DoxyInlineImage}
 ~\newline
Dort können alle verfügbaren Libraries durch sucht werden und dann mit {\ttfamily Add to Project} zu einem bestimmten Projekt hinzugefügt werden. Alle nötigen Einträge in die \mbox{\hyperlink{platformio_platformioini}{Konfiguration mit platformio.\+ini}} platformio.\+ini Datei werden automatisch gemacht. ~\newline
Es gibt jedoch eine Library, die dort nicht zu finden sein wird, die {\ttfamily Modellbahn} Library. Der Grund dafür ist, dass es unsere eigene ist, die wir nicht veröffentlicht haben. Um diese nutzen zu können, müssen wir dann doch in die \mbox{\hyperlink{platformio_platformioini}{Konfiguration mit platformio.\+ini}} platformio.\+ini Datei und Folgende Zeile einfügen\+:
